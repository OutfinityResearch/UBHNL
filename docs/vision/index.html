Why this system exists

UBHNL is built to give explicit, checkable meaning to text and rules. It complements LLMs by adding a deterministic layer that stores knowledge in CNL/DSL, detects contradictions, and returns evidence (origins, cores, witnesses).

Limits and accepted constraints

  • Deterministic input: CNL/DSL reject ambiguity; unknown symbols are errors.
  • Explicit modeling: the system does not infer missing semantics; the vocabulary and rules must be declared.
  • Finite domains: CNL/DSL is compiled by enumeration or schemas; very large domains require schema tactics.
  • Checkable results: UNSAT/OPTIMAL are accepted only with certificates; otherwise UNKNOWN.
  • Not a text generator: UBHNL complements LLMs; it does not replace them.

How the vision pages connect

Every domain page uses the same backbone: CNL/DSL as a precise meaning layer, strict vocabulary for typing, and checkable results with traceable origins. The Integration Map connects the domain workflows into a single System 2 stack.

What we still need

  • Domain-focused CNL templates and vocabulary libraries.
  • Extraction/annotation workflows to translate text into CNL safely.
  • More certificate checkers for non-Boolean backends.
  • Evaluation suites for consistency, coverage, and audit quality.
