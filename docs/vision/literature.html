Detect internal contradictions, track timelines, and enforce style constraints.

What this enables

  • Consistency checking: detect contradictions in character attributes, locations, and timelines.
  • Story logic constraints: enforce narrative rules (e.g., constraints on causality or knowledge).
  • Style rules: formalize constraints such as tense consistency, viewpoint, or repetition limits.

Workflow (practical)

  1. Define a narrative vocabulary: characters, locations, events, time points.
  2. Translate key facts and events into CNL statements.
  3. Add style constraints as rules (e.g., point-of-view restrictions).
  4. Query for inconsistencies (UNSAT cores show which statements conflict).

Outputs you can expect

  • Conflict reports with explicit explanations and origins.
  • Consistency proofs for subsets of the narrative.
  • Rule violations for style constraints.

Limits and constraints

  • Natural language text must be mapped to CNL; ambiguity is not accepted.
  • The system does not infer literary meaning beyond what is encoded.
  • Large timelines require schema tactics; results may return UNKNOWN under budget.

What we still need

  • Authoring tools for narrative extraction into CNL.
  • Reusable story schemas (time, location, possession, causality).
  • Evaluation suites for contradiction coverage and precision.
