Synthetic Data Generation

Generate data that satisfies explicit constraints and probabilistic weights.

What this enables

  • Constraint-respecting samples: models are only accepted if all rules hold.
  • Probabilistic control: weights and queries guide the distribution.
  • Auditability: generated data can be traced to the constraints it satisfies.

Workflow (practical)

  1. Encode data constraints and schema in CNL/DSL.
  2. Attach weights to literals for probabilistic objectives.
  3. Ask for models or samples; validate with the checker.
  4. Export witnesses as structured datasets.

Outputs you can expect

  • Concrete model assignments as synthetic records.
  • Distribution summaries with exact or approximate provenance.
  • Counterexamples when constraints are inconsistent.

Limits and constraints

  • Sampling quality depends on backend capability and fragment size.
  • Exact inference may be expensive; approximate results are clearly labeled.

What we still need

  • Sampling backends and export adapters for datasets.
  • Evaluation metrics for distribution fidelity.
  • Domain templates for common synthetic data tasks.
