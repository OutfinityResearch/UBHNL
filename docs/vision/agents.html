Agents and Tool Usage

Plan actions with explicit preconditions, policies, and safety constraints.

What this enables

  • Explicit plans: preconditions and postconditions as formal rules.
  • Policy enforcement: agents only execute actions that satisfy constraints.
  • Traceable decisions: every action has a rationale and origin.

Workflow (practical)

  1. Define action schemas (preconditions, effects, costs).
  2. Represent environment state and constraints in CNL/DSL.
  3. Query for feasible plans or validate LLM-generated plans.
  4. Use explanations and cores when plans fail.

Outputs you can expect

  • Valid action sequences (witnesses) with justifications.
  • Constraint-violating steps identified with origins.
  • Plan alternatives under different budgets or policies.

Limits and constraints

  • Planning quality depends on action schema completeness.
  • Complex temporal constraints may require dedicated fragments or backends.

What we still need

  • Action libraries for common tool APIs.
  • Cost and utility models to guide optimization.
  • Integration hooks for runtime tool execution.
