UNSAT core

A subset of constraints that is already unsatisfiable (used for explain/debug).

DRAFT: Wiki page.
Wiki Origin Certificate DS-009

Definition

An UNSAT core is a subset of constraints C ⊆ T such that C is UNSAT. In UBHNL, cores should preferably be reported at the origin level (statement-level), not as internal clause ids.

“Minimal” vs “minimal-ish”

  • Minimal core: removing any constraint makes it SAT (expensive to compute).
  • Minimal-ish: a small core obtained heuristically (often sufficient for practical explanations).

Role in explanations

  • For inconsistencies: “these 3 rules from files X/Y are in conflict”.
  • For entailment: the core for T ∧ ¬φ describes “why φ is derived”.

Relationship to certificates

An UNSAT core is not a full proof of UNSAT. The proof comes from a checkable certificate (e.g., DRAT/LRAT). The core is an explainability artifact.
