GoalKind

Intent classification: what the user wants (satisfy/prove/refute/count/optimize/synthesize/explain).

DRAFT: Wiki page.
Wiki Fragment Orchestrator DS-010

Definition

GoalKind captures “what kind of answer” the user requests. The same theory can be used for multiple goal kinds (e.g., satisfiability, entailment, counting).

Why this separation exists

  • The planner can choose different backends for the same fragment, depending on goal kind (e.g., KC for CountModels).
  • Certificate policy differs: “prove” requires UNSAT certificates; “satisfy” requires witnesses; “optimize” requires bound proofs.
