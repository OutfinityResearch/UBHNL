SyGuS (Syntax-Guided Synthesis)

A synthesis problem format where the solution must satisfy both semantics and a grammar.

Definition

Syntax-Guided Synthesis (SyGuS) is a family of synthesis problems where you must construct an artifact (usually an expression or program) that:

  1. meets a semantic specification (constraints must hold), and
  2. is built only from a given syntactic space (a grammar / template).

The grammar is not just a convenience: it is part of the problem statement. It restricts the search space and can encode domain knowledge (“only use these building blocks”).

How UBHNL treats synthesis

UBHNL models synthesis as a first-class goal kind (Synthesize) and expects a backend/tactic to return:

  • a synthesized artifact (e.g., a circuit, expression, or invariant), and
  • a checkable correctness justification (usually by reducing to SAT/SMT/UBH obligations).

In other words: the system may use advanced heuristics to find a candidate, but acceptance requires that the candidate is verifiable.

Toy example

Suppose you want to synthesize a Boolean expression f(x,y) that matches a truth table, but you restrict syntax to: and, or, not, and variables.

A SyGuS solver searches expressions in that grammar and returns a candidate like and(x,y). UBHNL then checks it by compiling the equivalence ∀x,y: f(x,y) ↔ spec(x,y) to a checkable backend.
