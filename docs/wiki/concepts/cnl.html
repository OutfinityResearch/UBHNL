CNL (Controlled Natural Language)

A deterministic subset of natural language, mapped 1-to-1 to a typed logic AST.

DRAFT: Wiki page.
Wiki Docs: Languages Lexicon Typed AST DS-005

Definition

CNL is a controlled language (near-natural language) with a small grammar and strict rules, such that parsing and typing are deterministic and do not rely on heuristics.

Non-negotiable rules

  • Explicit scope: explicit quantifiers; no pronouns/coreference.
  • Mandatory typing: each predicate has fixed arity and argument types (from the lexicon).
  • Ambiguity rejected: if two parses exist, it is an error (require parentheses/clarification).
  • Finite domains for lowering into UBH (for infinite domains: use another fragment/backend).

Example

For any Cell c:
    If c has GeneA and c does not have Inhibitor then c has ProteinP.

This is transformed into a typed AST and then compiled (possibly with the schema engine) into UBH constraints.

Why not “free-form NL”

  • Free-form NL introduces ambiguity and heuristics → results become hard to audit and reproduce.
  • UBHNL emphasizes stable specs and proof/counterexample workflows — therefore input must be deterministic.

Consequences

  • Invalid input produces a typed error (Lex/Parse/Resolve/Type), not a “guess”.
  • The CNL front-end is a compiler: it runs as part of the pipeline (not “AI”).
