Synthesis (SyGuS / invariant synthesis)

Not just “find a model”: construct an artifact (program/circuit/invariant) that satisfies a specification.

DRAFT: Wiki page.
Wiki GoalKind Certificate DS-011

Definition

Synthesis asks the system to construct an object (artifact) that satisfies a property: e.g., a circuit that implements a relation, an inductive invariant, a small program that passes tests.

In UBHNL

  • GoalKind: Synthesize. Typical fragments: TS/CHC/SMT/UBH.
  • Output: an artifact plus correctness obligations checked by SAT/SMT/UBH.

Certificate

The certificate is the artifact plus evidence that it discharges the obligations: a set of checkable queries (UNSAT/SAT with witness/cert).

Consequences

  • Requires a serializable artifact format (e.g., AST, circuit, invariant) and a checker for the obligations.
  • The orchestrator may use heuristics (including ML guidance), but acceptance depends on checkers.
