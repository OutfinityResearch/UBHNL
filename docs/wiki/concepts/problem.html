Problem (normalized query)

The standard internal form: theory + (optional) query formula + goal kind + options/budget.

DRAFT: Wiki page.
Wiki Fragment GoalKind Budget DS-010

Definition

In UBHNL, user input (CNL/DSL/AST) is normalized into a Problem: 

P = ⟨T, φ?, kind, options⟩
  • T: current theory (files + session delta)
  • φ?: query formula (optional, e.g., Prove/Refute)
  • kind: GoalKind
  • options: budgets, trust requirements, etc.

Why normalization

  • Backends have a common interface: they receive Problem + Budget.
  • The planner can compare tactics over the same representation.
  • Reproducibility: Problem + metadata → replay.
