Problem (query normalizat)

Forma internă standard: teorie + (opțional) formula query + goal kind + opțiuni/buget.

Definiție

În UBHNL, input-ul userului (CNL/DSL/AST) este normalizat la un Problem: 

P = ⟨T, φ?, kind, options⟩
  • T: teoria curentă (fișiere + session delta)
  • φ?: formula query (opțional, ex. Prove/Refute)
  • kind: GoalKind
  • options: bugete, trust requirements, etc.

De ce normalizare

  • Backends au o interfață comună: primesc Problem + Budget.
  • Planner-ul poate compara tactici pe aceeași reprezentare.
  • Reproducibilitate: Problem + metadata → replay.
