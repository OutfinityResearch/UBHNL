Knowledge compilation (BDD/d-DNNF/SDD)

Compile a Boolean theory into a representation that answers repeated queries quickly (prove/count/conditionals).

DRAFT: Wiki page.
Wiki #SAT/WMC Certificate DS-011

Definition

Knowledge compilation (KC) transforms a Boolean formula/theory into a structure (BDD, d-DNNF, SDD) where certain operations become cheap: repeated satisfiability checks, model counting, conditional queries.

In UBHNL

  • KC is treated as a fragment/artifact (Frag_KC) used as an accelerator for CountModels and repeated queries.
  • The planner can choose a tactic: “CompileKC → Check(equivalence) → AnswerQueriesFast”.

Certificate

When a backend produces a KC object, UBHNL requires a check of equivalence between the original theory and the compiled form. Typically: reduce equivalence checking to SAT/UBH.

Consequences

  • KC is useful when you run many queries over the same theory (session caching).
  • Upfront compilation cost can be high; budgets and heuristics are part of the orchestrator.
