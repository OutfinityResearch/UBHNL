XOR linear (GF(2) equations)

The linear fragment of UBH: equations of the form x1 ⊕ x2 ⊕ ... = b, solved via Gaussian elimination.

DRAFT: Wiki page.
Wiki GF(2) SAT DS-002

Definition

A XOR-linear equation has the form: 

x1 ⊕ x2 ⊕ ... ⊕ xk = b   (b ∈ {0,1})

It is “linear” in the sense of algebra over GF(2). Such systems are solved efficiently with Gaussian elimination.

A tiny example (solving by elimination)

Consider the system: 

x ⊕ y = 1
y ⊕ z = 0

From the second equation we get z = y. Substitute into the first: x ⊕ y = 1. If we additionally learn y = 0, then x = 1 and z = 0.

A Gaussian-elimination implementation does this mechanically using row operations over GF(2).

What maps here in UBH

  • XOR gate definitions: z = xor(a,b) becomes z ⊕ a ⊕ b = 0.
  • ASSERT_EQ(a,b) becomes a ⊕ b = 0.
  • ASSERT0/ASSERT1 become unit equations: w=0 / w=1.

Why not encode XOR into “pure” CNF

  • CNF encodings for XOR can be large and can destroy propagation.
  • Incremental Gauss provides strong, cheap propagation for circuits with many XORs.

Interaction with SAT

UBHNL uses a hybrid model: 

CNF side (AND via Tseitin)  ⟷  XOR side (linear equations)
         unit propagation            Gauss propagation
         decisions/learning          implications/conflicts

Certificates / checking

  • XOR implications can be verified by substitution into equations.
  • For global UNSAT, UBHNL prefers SAT-level certificates (DRAT/LRAT) after reduction, or XOR-aware proof objects when available.
