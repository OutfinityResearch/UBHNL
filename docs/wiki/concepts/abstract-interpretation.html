Abstract interpretation

A tactic/backend for invariants: over-approximation in abstract domains plus checkable proof obligations.

DRAFT: Wiki page.
Wiki Model checking CHC Certificate DS-011

Definition

Abstract interpretation derives invariants by interpreting a program/system in an abstract domain (e.g., intervals, octagons, polyhedra) that over-approximates the reachable states.

In UBHNL

  • It is a tactic/backend for Frag_AI and for invariant generation in TS/CHC workflows.
  • The useful output is an invariant; acceptance requires checking proof obligations via SAT/SMT/UBH.

Certificate

The certificate is the invariant plus evidence that it satisfies the obligations (e.g., Init⇒Inv, Inv∧Trans⇒Inv').
