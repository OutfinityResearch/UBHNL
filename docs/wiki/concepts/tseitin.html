Tseitin transformation

Linear-size CNF encoding for circuits: introduces auxiliary variables per sub-expression/gate.

DRAFT: Wiki page.
Wiki CNF UBH DS-002

The idea

Tseitin transforms an expression/circuit into an equisatisfiable CNF formula by introducing one variable per gate (sub-expression) and adding local constraints “output ↔ operator(inputs)”.

Example: AND gate

For z ↔ (a ∧ b), add the clauses: 

(¬a ∨ ¬b ∨ z)
(a ∨ ¬z)
(b ∨ ¬z)

Result: the number of clauses grows linearly in the number of gates (it does not explode exponentially).

Equisatisfiable (not necessarily equivalent)

Tseitin encodings usually introduce auxiliary variables (one per gate). That means the CNF is typically equisatisfiable with the original formula: it has the same SAT/UNSAT answer, but the set of models over all variables is different because auxiliaries can appear.

UBHNL relies on this because solve/prove needs only SAT/UNSAT correctness. For explanations, UBHNL maps models back to user-level atoms via the session’s symbol table and origins.

Why UBHNL does this

  • UBH is a circuit; Tseitin is the standard method to reach CNF efficiently.
  • It keeps a clean split: XOR stays in the linear subsystem (GF(2)), while AND becomes CNF.

Consequences

  • CNF becomes a derived artifact from the IR (it can be cached and reused in a session).
  • SAT models include auxiliary variables; for explanations we primarily care about “semantic” variables (predicate instances) mapped in the session.
