Lexicon (vocabulary + types)

The source of truth for symbols: finite domains, constants, predicates, and aliases.

DRAFT: Wiki page.
Wiki Docs: Languages CNL DSL DS-005

Definition

The lexicon (in UBHNL) is the vocabulary contract that defines types (domains), constants, and predicates. Vocabulary is derived from Sys2 schema files (*.sys2) and CNL domain files (*.cnl).

Why it is a “source of truth”

  • Determinism: same input + same lexicon ⇒ same semantic IR.
  • Without a lexicon there is no robust typing; without typing, “CNL/DSL” become ambiguous and fragile.
  • Safety policy: an unknown symbol is an error, not “implicit creation”.

Schema example

@Cell:Cell __Atom
@c0:c0 __Atom
@c1:c1 __Atom
IsA c0 Cell
IsA c1 Cell

Invariants (what the loader validates)

  • Referenced types exist (in domains or builtins, where applicable).
  • Constants have valid types and belong to their domains (or the domain is treated as authoritative).
  • Predicates have consistent arity and argument type lists (args).
  • Aliases do not collide with canonical names and are resolved deterministically.

Reserved keywords

CNL/DSL reserve keywords (e.g., ForAll, Exists, graph, return, end, And, Or, Not, Implies). For predictability, the lexicon should avoid using them as identifiers.

Engineering consequences

  • Theories (CNL/DSL files) become “compilable” and auditable: every token has a typed meaning.
  • Explanations can be printed in the user vocabulary: “ProteinP c0”, not “wire 123”.
