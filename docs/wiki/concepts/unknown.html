Status: UNKNOWN

An explicit result when no checkable evidence can be produced within the given budget.

DRAFT: Wiki page.
Wiki Decision: UNKNOWN Certificate Vision and Core Spec

Definition

UNKNOWN means: “the system cannot produce (within budget) an acceptable, checkable result”. It is not an error; it is a risk-control status.

When it happens

  • timeout / budget exceeded (time/memory/iterations),
  • a backend returned UNSAT or OPTIMAL without an acceptable certificate,
  • a probabilistic method without parameters/audit metadata (e.g., approximate model counting) or without sufficient trust.

Why it is preferred

  • UBHNL is audit-oriented: an unverifiable claim is worse than “I don’t know yet”.
  • It enables tactics/fallbacks: the orchestrator can try another backend or another reduction.

Practical consequences

  • APIs (solve/prove/query) must include UNKNOWN as a first-class status.
  • The result must include “audit metadata”: why UNKNOWN happened (which limit, which certificates are missing).

Relevant specs

  • DS-012 (unknown handling)
