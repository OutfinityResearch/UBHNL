Structural decompositions (treewidth / DP)

An orchestrator tactic: decompose the problem and solve subproblems via dynamic programming.

DRAFT: Wiki page.
Wiki Orchestrator Certificate DS-012

Definition

Some Boolean problems have an underlying graph structure with small treewidth. You can build a tree decomposition and then solve via dynamic programming on the tree.

In UBHNL

  • It is an orchestrator tactic: produces a “divide and conquer” plan.
  • The final output is checked via recomposition plus certificates for subproblems.
