Skolem Functions

Strategy objects that witness existential choices under universal quantifiers (FOL/QBF).

DRAFT: This page explains “Skolem functions” as an engineering-friendly certificate idea. QBF integration details are in DS-011/DS-014.
Wiki Quantifiers QBF Certificates DS-011

Problem context

Quantifiers introduce “choices”. In a formula like: 

∀u ∈ {0,1} : ∃e ∈ {0,1} : Φ(u, e)

the existential variable e is allowed to depend on u. The formula is satisfiable if, for every possible u, there exists a choice of e that makes Φ true.

Definition (engineering view)

A Skolem function is a function that makes those existential choices explicit: 

e = f(u)

If you can provide such a function f and it satisfies the formula for all u, then f is a checkable certificate of satisfiability.

Toy example

Consider: 

∀u : ∃e : (e ↔ u)

This is satisfiable with the Skolem function e = u. A checker verifies it by enumerating all values of u (here only 0 and 1) and checking that (e ↔ u) holds for e=f(u).

How UBHNL uses Skolem functions

UBHNL’s QBF fragment expects that a backend that claims “SAT” can provide a strategy. For many QBF formats, this can be represented as Skolem functions (often encoded as circuits). A trusted checker can validate the strategy by evaluating the QBF formula against the strategy.
