SAT (CDCL)

Boolean satisfiability solving: CDCL (clause learning) as the primary backend for CNF.

Definition

The SAT problem asks: given a Boolean formula (typically in CNF), does there exist a variable assignment that makes it true? CDCL (Conflict-Driven Clause Learning) is the modern family of algorithms that combine decisions, propagation, conflict analysis, and clause learning.

How CDCL works (engineer-level sketch)

  1. Decide: pick a variable and assign a value (a guess).
  2. Propagate: apply unit propagation on clauses; this deduces forced assignments.
  3. Conflict: if a clause becomes false, analyze the implication graph.
  4. Learn: add a new clause that prevents repeating the same conflicting combination.
  5. Backtrack: jump back to a safe decision level and continue.

This process either finds a satisfying assignment (SAT) or proves that none exists (UNSAT) and emits a certificate (typically DRAT/LRAT).

Role in UBHNL

  • UBH encodes AND gates into CNF (Tseitin); the CNF is solved with CDCL.
  • UBHNL may include an internal reference SAT backend or an adapter to an external solver.
  • CDCL cooperates with the XOR-linear engine (bidirectional propagation) — see the SAT+XOR decision.

Certificate

  • SAT: a checkable witness model (evaluate all clauses).
  • UNSAT: a checkable DRAT/LRAT certificate — see Certificates.

Engineering consequences

  • The solver interface must include UNKNOWN (timeout/budget/missing certificate).
  • Explanations (unsat cores) require mapping clauses/equations → origins, not just “clause ids”.
