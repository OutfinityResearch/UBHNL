SAT (CDCL)

Boolean satisfiability solving: CDCL (clause learning) ca backend principal pentru CNF.

Definiție

Problema SAT: dată o formulă booleană (de obicei în CNF), există o atribuire de variabile care o face adevărată? CDCL (Conflict-Driven Clause Learning) este familia de algoritmi moderni care combină: decizii, propagare, analiză de conflict și învățare de clauze.

Rol în UBHNL

  • UBH encodează AND-urile în CNF (Tseitin); CNF-ul este rezolvat cu CDCL.
  • UBHNL poate avea un backend SAT intern (de referință) sau adaptor către un solver extern.
  • CDCL cooperează cu motorul XOR-liniar (propagare bidirecțională) — vezi decizia SAT+XOR.

Certificate

  • SAT: witness model verificabil (clauzele sunt evaluate).
  • UNSAT: certificat DRAT/LRAT verificabil — vezi Certificate.

Consecințe inginerești

  • Interfața solverului trebuie să includă UNKNOWN (timeout/budget/certificat lipsă).
  • Explicațiile (unsat core) necesită mapare clauze/equations → origins, nu doar “clause ids”.

Spec-uri relevante

  • DS-002 (SAT backend pluggable + loop mixt)
  • DS-011 (DRAT/LRAT + acceptare)
