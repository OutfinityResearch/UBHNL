Trusted core

The minimal component you trust: deterministic checkers + the UBH evaluator.

DRAFT: Wiki page.
Wiki Decision: minimal kernel Certificate DS-011

Definition

The trusted core is the set of components whose correctness is assumed by the system. In UBHNL, it must be as small and as simple as possible.

Minimal contents

  • UBH evaluator/checker: verifies that a model satisfies UBH constraints.
  • Certificate checkers: DRAT/LRAT (SAT), invariants (TS), dual certs (MIP/SDP), etc.

What does NOT belong

  • complex solvers (SAT/SMT/IC3/MIP solvers) — these are untrusted by default and validated via certificates.
  • heuristics/ML guidance.

Consequences

  • Result acceptance becomes mechanical: check(result) == ok or UNKNOWN.
  • You can change backends without changing the system’s “truth”.
