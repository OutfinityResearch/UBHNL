Trusted core

Componenta minimă în care ai încredere: checkere deterministe + evaluator UBH.

DRAFT: pagină de Wiki.
Wiki Decizie: kernel minimal Certificate DS-011

Definiție

Trusted core este setul de componente a căror corectitudine este presupusă de sistem. În UBHNL, el trebuie să fie cât mai mic și cât mai simplu.

Conținut minim

  • UBH evaluator/checker: verifică că un model satisface constrângerile UBH.
  • Checkere de certificate: DRAT/LRAT (SAT), invariants (TS), dual certs (MIP/SDP), etc.

Ce NU intră

  • solvere complexe (SAT/SMT/IC3/MIP solvers) — acestea sunt backends “untrusted” prin default, validate prin certificate.
  • euristici/ML guidance.

Consecințe

  • Acceptarea rezultatelor devine mecanică: check(result) == ok sau UNKNOWN.
  • Poți schimba backends fără să schimbi “adevărul” sistemului.

Spec-uri relevante

  • DS-011 (trusted core)
  • System Spec (principii)
