Cuantificatori (∀ / ∃) pe domenii finite

Semantica, edge cases și strategiile de compilare (expandare vs schema engine).

DRAFT: pagină de Wiki.
Wiki Schema engine Type lowering DS-003

Definiție

În UBHNL, cuantificatorii din CNL/DSL sunt bine definiți atunci când domeniul este finit și enumerabil. Compilarea către UBH folosește această finitudine.

Semantica (cu edge cases)

forall x in D: P(x)  ≡  ∧_{d∈D} P(d)
exists x in D: P(x)  ≡  ∨_{d∈D} P(d)

Empty domain:

forall x in ∅: P(x)  == true
exists x in ∅: P(x)  == false

Strategii de implementare

  • Expandare completă: domenii mici; compilezi direct ∧/∨ de instanțe.
  • Schema engine: domenii mari; păstrezi regula ca schemă și instanțiezi la nevoie (DS-004).
  • Exists cu witness: pentru query-uri, preferi holes ca să obții witness fără disjuncție uriașă.

Consecințe inginerești

  • Parser/typing trebuie să cunoască domeniul și să respingă cuantificare fără domeniu.
  • Planner-ul trebuie să decidă expandare vs schema pe baza dimensiunii domeniului și a goal kind.

Spec-uri relevante
