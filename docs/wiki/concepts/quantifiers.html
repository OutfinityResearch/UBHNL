Quantifiers (∀ / ∃) over finite domains

Semantics, edge cases, and compilation strategies (expansion vs schema engine).

DRAFT: Wiki page.
Wiki Schema engine Type lowering DS-003

Definition

In UBHNL, quantifiers in CNL/DSL are well-defined when the domain is finite and enumerable. Compilation to UBH relies on this finiteness.

Semantics (with edge cases)

forall x in D: P(x)  ≡  ∧_{d∈D} P(d)
exists x in D: P(x)  ≡  ∨_{d∈D} P(d)

Empty domain:

forall x in ∅: P(x)  == true
exists x in ∅: P(x)  == false

Implementation strategies

  • Full expansion: small domains; compile directly into a ∧/∨ of instances.
  • Schema engine: large domains; keep the rule as a schema and instantiate on demand (DS-004).
  • Exists with a witness: for queries, prefer holes to obtain a witness without a huge disjunction.

Engineering consequences

  • Parsing/typing must know the domain and reject quantification without a domain.
  • The planner must decide expansion vs schema based on domain size and goal kind.
