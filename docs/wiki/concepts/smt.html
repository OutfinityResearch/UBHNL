SMT (Satisfiability Modulo Theories)

Generalizează SAT: booleeni + teorii (bitvectors, LIA, EUF) cu DPLL(T).

Wiki Fragment Certificate DS-010

Definiție

SMT rezolvă satisfiabilitatea pentru formule booleene construite din atomi care aparțin unei teorii (ex. aritmetică). Arhitectura clasică este DPLL(T): SAT decide structura booleană, iar “T-solvers” decid consistența atomilor.

În UBHNL

  • SMT este un backend pentru fragmente precum Frag_SMT_BV, Frag_SMT_LIA, Frag_SMT_EUF.
  • Pentru BV, o rută checkable frecventă este bit-blast → UBH/SAT + certificat SAT.
  • Pentru LIA, certificate pot veni ca dual/Farkas când e redus la LP/MIP; altfel, proof objects SMT.

Certificate

  • SAT: model SMT verificabil prin evaluarea atomilor.
  • UNSAT: proof object (ex. Alethe) sau reducere checkable (bit-blast + DRAT/LRAT).

Consecințe inginerești

  • Orchestratorul trebuie să știe când să păstreze problema nativ SMT și când să o coboare la UBH (cost/beneficiu).
  • Mapping-ul modelului trebuie “decodat” pentru explicații (ex. BV8 → integer/hex), păstrând origins.
