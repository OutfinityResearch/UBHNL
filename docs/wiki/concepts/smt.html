SMT (Satisfiability Modulo Theories)

Generalizes SAT: Booleans + theories (bitvectors, LIA, EUF) via DPLL(T).

DRAFT: Wiki page.
Wiki Fragment Certificate DS-010

Definition

SMT solves satisfiability for Boolean combinations of theory atoms (e.g., arithmetic atoms). The classic architecture is DPLL(T): SAT decides the Boolean structure, while “T-solvers” decide consistency of the theory atoms.

In UBHNL

  • SMT is a backend for fragments such as Frag_SMT_BV, Frag_SMT_LIA, Frag_SMT_EUF.
  • For BV, a common checkable route is bit-blast → UBH/SAT plus a SAT certificate.
  • For LIA, certificates can come as dual/Farkas witnesses when reduced to LP/MIP; otherwise, SMT proof objects.

Certificates

  • SAT: an SMT model checkable by evaluating theory atoms.
  • UNSAT: a proof object (e.g., Alethe) or a checkable reduction (bit-blast + DRAT/LRAT).

Engineering consequences

  • The orchestrator must decide when to keep a problem natively in SMT and when to lower it to UBH (cost/benefit).
  • Model mapping must be “decoded” for explanations (e.g., BV8 → integer/hex) while preserving origins.
