Budget (time/memory/iteration)

Explicit limits for solving/planning; part of reproducibility and the UNKNOWN policy.

DRAFT: Wiki page.
Wiki UNKNOWN Orchestrator DS-012

Definition

A budget limits the resources available for a query: time, memory, iterations (CEGAR), and sometimes method-specific parameters (e.g., seeds/confidence for probabilistic methods).

Why it is part of the contract

  • Explains why you can receive UNKNOWN even if the problem has an answer (it was not found within budget).
  • Enables replay: same plan + same budgets/seeds → same execution (subject to backend determinism).

Consequences

  • The orchestrator must split the budget across tasks (solve/check/refine).
  • Backends must be able to return “partial” results without claiming SAT/UNSAT/OPTIMAL.
