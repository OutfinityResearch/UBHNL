CP (Constraint Programming, finite domains)

Solving over finite domains with propagation + search (alldifferent, table constraints, etc.).

DRAFT: Wiki page.
Wiki Fragment Certificate DS-011

Definition

Finite-domain Constraint Programming represents variables with values in a finite set and global constraints (e.g., alldifferent) that propagate incrementally to shrink domains, plus backtracking search.

In UBHNL

  • Modeled as Frag_CP_FD: finite domains + CP constraints.
  • Can run natively (CP backend) or be lowered to SAT/BV/UBH for unification/checking.

Certificate

  • SAT: a variable assignment; check each constraint directly.
  • UNSAT: requires a proof log (if available) or translation to SAT/SMT with a certificate.

Consequences

  • CP is often excellent for scheduling/FD constraints; SAT can be better for large Boolean structure — hence the role of the orchestrator.
  • Good explanations require mapping constraints back to origins (not just “failed propagation”).
