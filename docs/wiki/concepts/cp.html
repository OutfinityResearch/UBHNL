CP (Constraint Programming, finite domains)

Rezolvare pe domenii finite cu propagare + search (alldifferent, table constraints, etc.).

DRAFT: pagină de Wiki.
Wiki Fragment Certificate DS-011

Definiție

Constraint Programming pe domenii finite reprezintă variabile cu valori într-un set finit și constrângeri globale (ex.: alldifferent) care se propagă incremental pentru a reduce domeniile, plus backtracking search.

În UBHNL

  • Modelat ca Frag_CP_FD: domenii finite + constrângeri CP.
  • Poate rula nativ (backend CP) sau poate fi “lowered” la SAT/BV/UBH pentru unificare/checking.

Certificate

  • SAT: asignare de variabile; verifici direct fiecare constrângere.
  • UNSAT: necesită proof log (dacă există) sau traducere la SAT/SMT cu certificat.

Consecințe

  • CP este adesea excelent pentru scheduling/FD constraints; SAT poate fi mai bun pentru booleeni masivi — de aici rolul orchestratorului.
  • Explicațiile bune cer “reificare” a constrângerilor în origins (nu doar “failed propagation”).
