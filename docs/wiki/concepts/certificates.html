Certificate (proof-carrying results)

Politica UBHNL: rezultatele sunt acceptate doar dacă pot fi verificate independent.

Definiție

Un certificat este un artefact de date care permite unui checker determinist să valideze afirmația unui backend: UNSAT, OPTIMAL, “invariant corect”, “strategie corectă”, etc.

Politica de acceptare

  • SAT: acceptat doar cu witness model verificabil.
  • UNSAT: acceptat doar cu certificat verificabil (sau confirmare printr-o rută checkable).
  • OPTIMAL: acceptat doar cu witness + dovadă de bound (dual/bounds/UNSAT-of-tightened).
  • Altfel: UNKNOWN (nu “ghicim”).

Certificate SAT: DRAT/LRAT (pe scurt)

Pentru CNF SAT, certificatele uzuale pentru UNSAT sunt trasee de rezoluție în formate precum DRAT/LRAT. Checker-ul verifică determinist că fiecare pas este valid și că se obține contradicție.

În UBHNL, asta este “ancora” pentru acceptarea UNSAT la nivel bit/UBH: dacă reducerea la CNF este corectă, DRAT/LRAT închide bucla.

Consecințe inginerești

  • Trebuie un “trusted core” mic: UBH evaluator + checkere de certificate (vezi DS-011).
  • Backends pot fi “untrusted/heuristic”, dar rezultatele lor nu sunt finale fără verificare.
  • Explicațiile pot reutiliza certificatele: “de ce UNSAT” = core + traseu.
