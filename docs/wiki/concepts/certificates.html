Certificate (proof-carrying results)

UBHNL policy: results are accepted only if they can be independently verified.

DRAFT: Wiki page.
Wiki Decision: certificates UNKNOWN DS-011

Definition

A certificate is a data artifact that allows a deterministic checker to validate a backend claim: UNSAT, OPTIMAL, “invariant is correct”, “strategy is correct”, etc.

Acceptance policy

  • SAT: accepted only with a checkable witness model.
  • UNSAT: accepted only with a checkable certificate (or confirmation via a checkable route).
  • OPTIMAL: accepted only with a witness plus a bound proof (dual/bounds/UNSAT-of-tightened).
  • Otherwise: UNKNOWN (no “guessing”).

SAT certificates: DRAT/LRAT (short)

For CNF SAT, common UNSAT certificates are resolution traces in formats such as DRAT/LRAT. The checker deterministically validates that each step is valid and that a contradiction is derived.

In UBHNL, this is the “anchor” for accepting UNSAT at the bit/UBH level: if the reduction to CNF is correct, DRAT/LRAT closes the loop.

Engineering consequences

  • You need a small “trusted core”: UBH evaluator + certificate checkers (see DS-011).
  • Backends may be untrusted/heuristic, but their results are not final without verification.
  • Explanations can reuse certificates: “why UNSAT” = core + trace.
