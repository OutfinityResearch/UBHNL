Symbolic execution

Program analysis: explore paths, generate path constraints, and use SMT/SAT to find counterexamples.

DRAFT: Wiki page.
Wiki SMT Certificate CHC

Definition

Symbolic execution runs a program on symbolic inputs while accumulating constraints (path constraints). When you want a bug/counterexample, you solve those constraints with SMT/SAT.

In UBHNL

  • Treated as a tactic/backend over SMT fragments: it generates SMT problems from code.
  • Main output: a counterexample (concrete input) or invariants/cores when proving absence of a bug on a bounded subset.

Certificates

  • Counterexample: witness input plus replay verification (re-execution/simulation).
  • Safety proofs: invariants plus obligations checked by SMT/SAT.
