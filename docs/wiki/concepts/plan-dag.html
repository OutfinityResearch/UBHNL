Plan DAG (task graph)

The execution plan representation: nodes = tasks; edges = dependencies.

DRAFT: Wiki page.
Wiki Orchestrator Backends DS-012

Definition

A plan DAG is an acyclic task graph that describes the full query pipeline: compilation/lowering, solving, checking, refinement, and explanation.

Task kinds (minimal)

  • Lower: compile semantic IR into a target fragment (e.g., BV â†’ UBH).
  • Solve: run a backend for a fragment.
  • Check: validate a witness/certificate in the trusted core.
  • Refine: a CEGAR step (instances/lemmas/invariants).
  • Explain: produce the final explanation (origins).

Why a DAG (not a list)

  • Some tasks can run in parallel (e.g., two candidate backends in a fallback).
  • You can reuse sub-tasks (caching, incremental sessions).
