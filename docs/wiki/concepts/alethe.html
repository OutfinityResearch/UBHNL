Alethe (SMT Proof Format)

An SMT proof-object format used to make UNSAT results independently checkable.

DRAFT: This page describes the intended role of Alethe in UBHNL. The exact accepted subset is specified in DS-014.
Wiki SMT Certificates Trusted core DS-011

Definition

Alethe is a machine-readable format for SMT proof objects. In practice, it is used as an evidence payload for an UNSAT claim: a separate checker can replay the proof and confirm that the input is indeed unsatisfiable, without re-solving the problem.

Why UBHNL mentions Alethe

UBHNL’s system-level rule is “do not trust solvers, verify results”. For purely Boolean problems we can validate UNSAT via DRAT/LRAT. For SMT problems, a comparable approach is required: an SMT backend should return a checkable proof object. Alethe is one practical candidate format for that role.

How it fits in the architecture

In UBHNL, Alethe is treated as an artifact attached to a backend result. The orchestrator accepts an SMT backend’s UNSAT only if a trusted checker validates the proof object (or if the problem is reduced to SAT/UBH with a Boolean proof).

UBHNL policy (summary)

  • SMT backends may return UNSAT with a proof object (e.g., Alethe), but it must be checked.
  • If proof checking is not available, the orchestrator must either confirm via a checkable reduction (e.g., bit-blast→SAT) or return UNKNOWN.
  • The exact supported proof subset and checker invocation are specified in DS-014.
