Wire / WireId

The identity of a UBH wire: a Boolean value evaluable in {0,1}.

DRAFT: Wiki page.
Wiki UBH UBH-SPEC DS-001

Definition

A wire is a Boolean value (bit) produced by a constant, an input variable, or a gate (XOR/AND). A WireId is an integer identifier that refers to that wire.

Contracts

  • Instance stability: within a UBH kernel instance, creating the same gate wire (same op and canonicalized children) returns the same id.
  • Ids are not persistent across runs: persistence is a session/theory-store responsibility.
  • Canonicalized commutativity: for XOR/AND, (a,b) are ordered to avoid duplicates.

Why it matters

  • WireId is the common “currency”: the solver, checker, and origin mapping all refer to these ids.
  • Explanations must reconstruct user terms (predicate instances) from ids, hence the session-level mappings.
