Witness Queries

Existential binders whose assignments are returned as witnesses.

DRAFT: Wiki page.
Wiki DSL Session DS-009

Definition

A witness query is an existential statement submitted as a query. The binder variable is returned in the result when a model exists.

How it is introduced

Which Cell c has ProteinP?

DSL form:

@q1 Exists Cell graph c
    @c1 ProteinP $c
    return $c1
end

What is returned

{
  "status": "SAT",
  "witnessAssignments": { "c": "c0" }
}

Difference from “normal” exists

  • The syntax is the same (an Exists ... graph v block).
  • In query context, the binder variable is returned as a witness assignment.

Consequences

  • The planner may choose different tactics (e.g., deterministic enumeration with budgets) to find a witness.
  • The explanation includes “why this witness” (model + relevant origins).
