Holes (?x)

Existential variables that are returned as witnesses (hole filling).

DRAFT: Wiki page.
Wiki DSL Session DS-009

Definition

A hole is an existential unknown that must be instantiated so that the query becomes satisfiable (i.e., a witness exists). In DSL notation: ?x.

How it is introduced

exists ?c in Cell: proteinP(?c)

?c is existential, but (unlike a standard $c) it is explicitly requested as output.

What is returned

{
  "status": "SAT",
  "holeAssignments": { "?c": "c0" }
}

Difference from “normal” exists

  • exists $x in D: ... is a logical formula; it does not necessarily ask you to expose the witness.
  • exists ?x in D: ... is an interface requirement: “return me a witness” (if one exists).

Consequences

  • The planner may choose different tactics (e.g., deterministic enumeration with budgets) to find a witness.
  • The explanation includes “why this witness” (model + relevant origins).
