Holes (?x)

Variabile existențiale “returnabile” ca witness (hole filling).

DRAFT: pagină de Wiki.
Wiki DSL Session DS-009

Definiție

Un hole este un necunoscut existențial care trebuie instanțiat astfel încât query-ul să fie satisfiabil (sau să existe un witness). Notarea în DSL: ?x.

Cum se introduce

exists ?c in Cell: proteinP(?c)

?c este existențial, dar (spre deosebire de un $c standard) este cerut explicit ca output.

Ce se întoarce

{
  "status": "SAT",
  "holeAssignments": { "?c": "c0" }
}

Diferență față de exists “normal”

  • exists $x in D: ... este o formulă logică; nu cere neapărat să expui witness-ul.
  • exists ?x in D: ... este o cerință de interfață: “dă-mi un witness” (dacă există).

Consecințe

  • Planner-ul poate alege tactici diferite (ex.: enumerare deterministă cu buget) pentru a găsi un witness.
  • Explicația include “de ce acest witness” (model + origins relevante).
