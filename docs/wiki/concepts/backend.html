Backend

A pluggable solver/engine that solves a fragment for a goal kind and returns checkable artifacts.

DRAFT: Wiki page.
Wiki Problem Certificate Trusted core DS-011

Definition

A backend is an implementation (internal or an adapter) that receives a Problem (plus a budget) and returns a normalized result: status + witness/certificate + artifacts (invariant, KC object, etc.).

Interface (conceptual)

solve(problem: Problem, budget: Budget) -> BackendResult

BackendResult (normalizat)

{
  status: "SAT"|"UNSAT"|"UNKNOWN"|...,
  model?: Model,
  artifact?: Artifact,
  certificate?: Certificate,
  core?: OriginId[]
}

Trust model

  • Backends can be “checkable” or “heuristic”, but acceptance goes through checkers.
  • A heuristic backend without confirmation → UNKNOWN for strong claims.
