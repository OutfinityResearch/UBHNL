Definiție

Un backend este o implementare (internă sau adaptor) care primește un Problem (plus buget) și întoarce un rezultat normalizat: status + witness/certificat + artefacte (invariant, KC object, etc.).

Interfață (conceptual)

solve(problem: Problem, budget: Budget) -> BackendResult

BackendResult (normalizat)

{
  status: "SAT"|"UNSAT"|"UNKNOWN"|...,
  model?: Model,
  artifact?: Artifact,
  certificate?: Certificate,
  core?: OriginId[]
}

Trust model

  • backend-urile pot fi “checkable” sau “heuristic”, dar acceptarea trece prin checkere.
  • un backend heuristic fără confirmare → UNKNOWN pentru afirmații tari.

Spec-uri relevante
