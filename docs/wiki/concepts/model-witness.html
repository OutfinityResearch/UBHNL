Model / Witness

What “SAT” means in a system that accepts only checkable results.

DRAFT: Wiki page.
Wiki Constraints Certificate DS-011

Definition

A model (a.k.a. a witness) is an assignment that makes a theory satisfiable. In UBH, a model assigns values to input variables (and, in practice, to all relevant wires) such that all constraints evaluate to true.

Why a witness is mandatory

  • SAT must be checkable: deterministically verify that all constraints are satisfied under the model.
  • Without a witness, “SAT” would be an unverifiable claim → unacceptable for audit.

Witnesses and witness queries

For witness queries (existential queries), the witness includes an assignment for the binder variable (e.g., a domain constant). It is still a “model”, but with a “returnable” component.
