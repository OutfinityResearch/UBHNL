Schema engine (lazy instantiation)

Tratează cuantificatori/rule generative fără grounding complet: instanțiază doar când modelul violează schema.

Definiție

Un schema engine păstrează axiome/rule cuantificate ca “scheme” (nu expandate), apoi folosește modele SAT ca să găsească violări și să genereze instanțe concrete.

Workflow (universal schemas)

schemas = { forall x in D: Body(x), ... }
instances = {}

loop:
  solve(T + instances)
    if UNSAT: stop
    if SAT: find violations under model M
      if none: fixed point (model satisfies all schemas)
      else add instances for violating assignments and repeat

Ce înseamnă “violare”

  • forall x in D: Body(x) e violată dacă există d ∈ D cu Body(d)=false în model.
  • Pentru regula Premise(x) -> Conclusion(x), violare = Premise(d)=true și Conclusion(d)=false.

Politici de instanțiere

  • minimal: adaugă 1 instanță per iterație (teorie mică, multe iterații).
  • batch: adaugă toate violările curente (iterații mai puține, creștere mai mare).

Spec-ul recomandă batch cu limită per iterație.

Exists: de ce e diferit

exists x in D: Body(x) e “violată” doar dacă toate instanțele sunt false. Opțiuni:

  • expandare completă (domenii mici),
  • căutare de witness prin holes (domenii mari),
  • creștere incrementală a disjuncției (advanced).

Domenii mutable

Dacă domeniile pot fi extinse în sesiune, instanțele trebuie invalidate sau “keyed” după un domain-generation id, ca să nu amesteci instanțe din universuri diferite.
