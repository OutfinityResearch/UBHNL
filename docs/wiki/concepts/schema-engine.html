Schema engine (lazy instantiation)

Handles quantifiers/generative rules without full grounding: instantiate only when a model violates the schema.

DRAFT: Wiki page.
Wiki CEGAR Quantifiers DS-004

Definition

A schema engine keeps quantified axioms/rules as “schemas” (not expanded), then uses SAT models to find violations and generate concrete instances.

Workflow (universal schemas)

schemas = { forall x in D: Body(x), ... }
instances = {}

loop:
  solve(T + instances)
    if UNSAT: stop
    if SAT: find violations under model M
      if none: fixed point (model satisfies all schemas)
      else add instances for violating assignments and repeat

What “violation” means

  • forall x in D: Body(x) is violated if there exists d ∈ D with Body(d)=false under the model.
  • For a rule Premise(x) -> Conclusion(x), a violation is Premise(d)=true and Conclusion(d)=false.

Instantiation policies

  • minimal: add 1 instance per iteration (small theory, many iterations).
  • batch: add all current violations (fewer iterations, larger growth).

The spec recommends batch mode with a per-iteration limit.

Exists: why it is different

exists x in D: Body(x) is “violated” only if all instances are false. Options:

  • full expansion (small domains),
  • witness search via holes (large domains),
  • incremental disjunction growth (advanced).

Mutable domains

If domains can be extended in a session, instances must be invalidated or keyed by a domain-generation id, so that you do not mix instances from different universes.
