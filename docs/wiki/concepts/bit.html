Bit / Boolean

The fundamental unit: values in {0,1} and truth semantics.

DRAFT: Wiki page (glossary). The normative spec remains in MD.
Wiki Theory Docs UBH-SPEC

Definition

A bit is an element of {0,1}. In UBHNL, a bit is also a Boolean: 0 means false, and 1 means true.

Why it matters in UBHNL

  • The UBH kernel has a single native type: Bit. Every other type (BV, enum, set, relation) is encoded as bits.
  • Solving and checking reduce to evaluating Boolean circuits under a bit assignment (a model).
  • Practical (finite) universality: any finite discrete structure can be encoded in bits.

Minimal notation

NotationMeaning
0false
1true
⊕XOR (addition modulo 2) — see GF(2)
∧AND (multiplication) — see GF(2)

Engineering consequences

  • Any “high-level” API must specify its bit encoding explicitly (see type lowering).
  • Checkers are simple and robust: for SAT, you validate “the model satisfies the constraints” by deterministic evaluation.
