Proof assistants (HOL / Type Theory)

Backends pentru dovezi formale: proof objects verificate de un kernel mic (trusted core).

Definiție

Un proof assistant (ex. HOL/Type Theory) construiește dovezi ca obiecte (proof terms) care pot fi verificate de un kernel mic, determinist.

Rol în UBHNL

  • Nu mută complexitatea în kernel UBH; rămâne un backend modular.
  • Poate gestiona fragmente foarte expresive (Frag_HOL, Frag_TypeTheory).
  • Integrare tipică: proof scripts ca artefacte long-term (theory files) + discharge automat de sub-goal-uri cu SAT/SMT.

Certificate

Certificatul este chiar proof object-ul. Checker-ul este kernel-ul proof assistant-ului (trusted core).

Consecințe

  • Acceptarea devine puternică (formal), dar costul în experiența userului poate fi mai mare (proof engineering).
  • UBHNL tratează asta ca “slot arhitectural”: nu e obligatoriu pentru nucleul UBH, dar e compatibil.

Spec-uri relevante

  • DS-010 (Frag_HOL, Frag_TypeTheory)
  • DS-011 (certificate proof assistants)
