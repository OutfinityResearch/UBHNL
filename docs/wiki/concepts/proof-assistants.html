Proof assistants (HOL / Type Theory)

Backends for formal proofs: proof objects checked by a small kernel (trusted core).

DRAFT: Wiki page.
Wiki Certificate Minimal kernel DS-011

Definition

A proof assistant (e.g., HOL/Type Theory) constructs proofs as objects (proof terms) that can be checked by a small, deterministic kernel.

Role in UBHNL

  • It does not move complexity into the UBH kernel; it remains a modular backend.
  • It can handle very expressive fragments (Frag_HOL, Frag_TypeTheory).
  • Typical integration: proof scripts as long-term artifacts (theory files) plus automated discharge of sub-goals via SAT/SMT.

Certificate

The certificate is the proof object itself. The checker is the proof assistant’s kernel (trusted core).

Consequences

  • Acceptance is very strong (formal), but the user experience cost can be higher (proof engineering).
  • UBHNL treats this as an architectural “slot”: not required for the UBH kernel, but compatible.

Relevant specs

  • DS-010 (Frag_HOL, Frag_TypeTheory)
  • DS-011 (certificate proof assistants)
