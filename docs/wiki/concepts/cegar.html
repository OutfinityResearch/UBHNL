CEGAR (Counterexample-Guided Abstraction Refinement)

Strategie de compresie și solving: începi cu o abstracție, apoi o rafinezi cu contraexemple.

DRAFT: pagină de Wiki.
Wiki Schema engine Model checking DS-004

Definiție

CEGAR este un loop: 

solve(abstraction)
  if UNSAT: proved (certificate)
  if SAT: analyze counterexample
        if counterexample is real: disproved (witness)
        else refine abstraction and repeat

În UBHNL: două cazuri principale

  • Schema engine: axiome cuantificate rămân ca scheme; când un model le violează, instanțiezi doar cazurile necesare.
  • Verificare TS: BMC produce trace-uri; dacă sunt spurious (din abstracție), rafinezi (lemmas/invariants).

De ce este “compresie”

  • Nu ground-uiești tot domeniul/rule set-ul upfront.
  • Teoria concretă crește doar cât e necesar pentru query-ul curent.

Consecințe

  • Trebuie definit clar ce înseamnă “contraexemplu real” vs “spurious” în fragmentul curent.
  • Loop-ul trebuie să aibă bugete/limite și să poată termina cu UNKNOWN.
