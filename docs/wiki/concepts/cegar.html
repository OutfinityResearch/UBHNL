CEGAR (Counterexample-Guided Abstraction Refinement)

A compression and solving strategy: start with an abstraction, then refine it using counterexamples.

DRAFT: Wiki page.
Wiki Schema engine Model checking DS-004

Definition

CEGAR is a loop: 

solve(abstraction)
  if UNSAT: proved (certificate)
  if SAT: analyze counterexample
        if counterexample is real: disproved (witness)
        else refine abstraction and repeat

In UBHNL: two main cases

  • Schema engine: quantified axioms remain as schemas; when a model violates them, instantiate only the necessary cases.
  • TS verification: BMC produces traces; if they are spurious (from an abstraction), refine (lemmas/invariants).

Why it is “compression”

  • You do not ground the entire domain/rule set upfront.
  • The concrete theory grows only as much as needed for the current query.

Consequences

  • You must define clearly what “real counterexample” vs “spurious” means in the current fragment.
  • The loop must have budgets/limits and be able to terminate with UNKNOWN.
