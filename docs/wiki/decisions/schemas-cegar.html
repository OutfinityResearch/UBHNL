Decision: schema engine (lazy instantiation) for quantifiers

Keep rules as schemas; instantiate only what is needed for a query (compression).

DRAFT: decision page.
Wiki Schema engine CEGAR DS-004

The decision

For large (but finite) domains, we do not expand forall/exists upfront into thousands/millions of instances. We keep axioms as schemas and instantiate them in a CEGAR loop.

Why

  • Without this, structural compression is lost (explosive grounding).
  • Solver models can guide exactly which instances are relevant.

Consequences

  • The query planner becomes responsible for budgets/iterations and may terminate with UNKNOWN.
  • Explanations include the generated instances (and the origin of the schemas).
