Decizie: schema engine (instanțiere lazy) pentru cuantificatori

Menține reguli ca scheme; instanțiază doar ce e necesar pentru query (compresie).

Wiki Schema engine CEGAR DS-004

Decizia

Pentru domenii mari (dar finite), nu expandăm upfront forall/exists în mii/milioane de instanțe. Păstrăm axiomele ca scheme și instanțiem în loop CEGAR.

De ce

  • Fără asta, compresia structurală se pierde (grounding exploziv).
  • Modelele solverului pot ghida exact ce instanțe sunt relevante.

Consecințe

  • Query planner devine responsabil pentru bugete/iterații și poate termina cu UNKNOWN.
  • Explicațiile includ instanțele generate (și originea schemelor).
