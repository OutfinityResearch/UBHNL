Decision: UNKNOWN is preferable to an unverifiable claim

Integrity policy: if you cannot verify it, you do not claim it is proved.

The decision

If a backend does not produce an acceptable witness/certificate within the budget, UBHNL returns UNKNOWN. There is no “silent fallback” to a strong claim.

Why

  • Protects against bugs and unaudited probabilistic methodologies.
  • Forces the system to be explicit about limits (budgets, missing certificates).

Consequences

  • The API must include metadata for UNKNOWN (why, what was attempted).
  • The planner may try multiple routes, but it cannot “accept” without verification.
