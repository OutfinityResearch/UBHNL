Example 1 — Propositional (UBH/SAT)

Goal: Prove: (A and B) implies C

Lexicon (minimal)

{
  "domains": {},
  "constants": {},
  "predicates": {
    "A": { "arity": 0, "args": [] },
    "B": { "arity": 0, "args": [] },
    "C": { "arity": 0, "args": [] }
  }
}

Theory file: t.dsl

A() implies C()
B() implies C()

Query

Prove: (A() and B()) implies C()

Expected

  • PROVED
  • certificate: UNSAT for T ∧ ¬φ (DRAT/LRAT or equivalent)
  • explain: points to the two implications as the relevant core

Example 2 — SMT(BV) (bit-blast or SMT proof)

Goal: a bitvector proof: x=255, x+1=y ⊨ y=0 (mod 256).

This example assumes a BV frontend (e.g., an SMT-LIB adapter) that produces semantic IR in Frag_SMT_BV and then either:

  • runs an SMT(BV) backend with a proof/model, or
  • bit-blasts into UBH/SAT and uses DRAT/LRAT for UNSAT/PROVED.

Query

Learn:
  x:BV8, y:BV8
  x = 255
  y = x + 1

Prove:
  y = 0

Expected

  • PROVED
  • certificate: SMT proof object or SAT certificate after bit-blasting

Example 3 — Transition system (BMC / IC3/PDR)

Goal: Prove: Always(s != 3) for a modulo-4 counter.

Model

State: s:BV2
Init:  s = 0
Trans: s' = s + 1 mod 4
Safe:  s != 3

Expected

  • DISPROVED
  • checkable counterexample trace: 0 → 1 → 2 → 3
  • explain: shows the transition steps and that s=3 violates Safe

Example 4 — Hole filling (witness)

Goal: find a ?c that satisfies a property.

Theory file: bio.dsl

@c0:Cell
@c1:Cell
@c0 proteinP

Query

Satisfy: exists ?c in Cell: proteinP(?c)

Expected

{
  "status": "SAT",
  "holeAssignments": { "?c": "c0" }
}
