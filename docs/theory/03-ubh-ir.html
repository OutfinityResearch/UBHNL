Theory 03 — UBH: IR, hash-consing, semantics

What the minimal kernel looks like, and why structural compression is “by construction”.

DRAFT: tutorial chapter.
Prev Theory Wiki Next

1) UBH as a DAG of gates

UBH represents expressions as a DAG of nodes (wire/nodeId) with XOR/AND gates and constants.

2) Hash-consing = structural compression

  • Each node is created once and then reused (hash-consing).
  • XOR/AND commutativity is canonicalized (input ordering) before hashing.
  • Local simplifications run before hashing to increase deduplication.

3) Semantics: evaluation under a model

Given a model (assignments for variables), each wire is evaluated recursively. The trusted core can validate a SAT claim by deterministic evaluation.

4) Constraints are separate from structure

UBH IR describes “how a wire is computed”. Constraints describe “what must be true”. This separation keeps the kernel small and makes composition predictable.
