Theory 05 — Lexicon, CNL, DSL

Why input must be deterministic and typed, and how we achieve that.

1) The lexicon is the system signature

The lexicon defines types/domains, constants, and predicates. Without it, CNL/DSL cannot be typed robustly, and “unknown word” handling becomes ambiguous.

2) CNL: natural language without ambiguity

  • CNL has a small, deterministic grammar.
  • Quantifiers and connectives are explicit; pronouns/coreference are rejected (or handled only by explicit user disambiguation).

3) DSL: a “subject-first” textual IR plus core logic

  • DSL is the preferred format for theory files (long-term memory).
  • The @/$/vocab rule (and “single @ per line”) makes parsing simple and reduces user mistakes.

4) Typing and useful errors

Strict-vocabulary policy (decision) means: unknown symbol ⇒ load error (with line/column), not “create a new symbol”.
