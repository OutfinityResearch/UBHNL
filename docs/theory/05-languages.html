Theory 05 — Lexicon, CNL, DSL

Why input must be deterministic and typed, and how we achieve that.

1) Natural CNL — English without ambiguity

CNL looks like natural English but parses deterministically. No parentheses needed for most statements: 

# Facts (natural SVO)
Alice trusts Bob.
p1 has Fever.
Socrates is Mortal.

# Rules (natural If-then)
For any Patient p:
    If p has Flu then p has Fever.
    If p has Cold then p coughs.

2) Supported natural patterns

CNL PatternExampleDSL Output

3) DSL — the machine-readable format

  • DSL is the storage format for theory files.
  • Uses @name for declarations, $name for references.
  • Example: @rule1 ForAll Person graph x ... end

4) Theory files replace JSON schemas

Domain knowledge is now expressed in .cnl files, not JSON: 

# theories/domains/medical.cnl
Patient is a Domain.
For any Patient p:
    If p has Flu then p has Fever.

See /theories/ for reusable domain theories.

5) Python-style indentation

Blocks are defined by indentation (4 spaces), not braces: 

For any Person x:
    x has Fever.      # Inside block
    x is sick.        # Also inside
x is healthy.         # Outside (dedented)
