The Use Case: The Compliance Officer

Sarah is a compliance officer at a large bank. She needs to ensure that thousands of transactions follow the law: "If a transaction is over $10,000 and the sender is from a high-risk region, then it must be flagged for review."

Sarah isn't a coder, but she is a logic expert. UBHNL gives her a Controlled Natural Language (CNL) that looks like English but acts like code.

CNL: English without the "Oops"

Human language is messy. "I saw the man with the telescope" (Did I use the telescope, or did he have it?). UBHNL uses Controlled Natural Language to remove this ambiguity. It uses strict patterns like:

  • X has Y (e.g., Transaction has HighRiskSender.)
  • X is Y (e.g., Alice is a Manager.)
  • If P then Q (e.g., If amount > 10000 then Flag(transaction).)
By following these patterns, Sarah can write rules that the computer understands exactly as she intended.

Two Skins, One Soul: CNL vs. DSL

While Sarah loves the CNL, developers might prefer something more structured. UBHNL provides an isomorphism: Every sentence in Sarah's English-like CNL has a 1-to-1 equivalent in our DSL (sys2).

CNL (Human)
Every Patient with Flu has Fever.
↔
DSL (Machine)
@rule1 ForAll Patient graph p
  Implies { HasFlu $p } { HasFever $p }
end

You can swap between them at any time. The system doesn't care which one you use—the underlying logic remains identical.

The Lexicon: The Common Dictionary

How does the system know what a "Patient" or "Flu" is? Through the Lexicon. Before you start writing rules, you define your "vocabulary". This ensures that the system is Strictly Typed. If Sarah mentions a "Symptom" that isn't in her lexicon, the system will politely stop her and ask for a definition.

Where we are going

What we have: A deterministic translator that handles SVO (Subject-Verb-Object) patterns and complex quantifiers.

What we need: We are working on Grammar Extensions. We want to support more natural ways of expressing time ("After a transaction is flagged...") and uncertainty ("It is 80% likely that...").
