Theory 06 — Session: memory, origins, explain

Why sessions exist, and how long-term files connect to short-term solving.

DRAFT: tutorial chapter.
Prev Theory Wiki Next

1) Long-term vs short-term

  • Theory files: on disk, reproducible, loaded deterministically.
  • Session: an ephemeral workspace that can accept transient learn deltas and cache artifacts.

2) Origins: “where did this come from?”

For explainability and audit, each statement produces an Origin. Unsat cores are reported in terms of origins, not internal nodes or solver-specific ids.

3) Holes: queries that return witnesses

The session tracks holes and returns holeAssignments as part of the result.

4) Caching and reproducibility

  • File digests are used as cache keys.
  • Reproducibility: the same files + deltas + seeds ⇒ the same plan and result (within backend determinism limits).
