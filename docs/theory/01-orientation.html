Ce e UBHNL, în propoziția de lucru

UBHNL este un sistem universal de reasoning în care: kernel-ul este un substrat boolean minimal (UBH), iar toate celelalte logici/metode sunt front-ends + backends orchestrate, cu rezultate acceptate doar cu certificate.

De ce “universal” fără monolit

  • Universalitatea e obținută prin encodări (type lowering) + orchestrare, nu printr-un singur solver gigantic.
  • Decizia kernel minimal păstrează trusted core mic și schimbabilitatea backends.

Straturi (model mental)

theory files (CNL/DSL) + learn (session)
        │
        ▼
parse/type (lexicon, strict vocab) → typed semantic IR
        │
        ▼
fragment detection + goal kind
        │
        ▼
orchestrator plan DAG: lower/solve/check/refine/explain
        │
        ▼
result (SAT/UNSAT/UNKNOWN, PROVED/DISPROVED) + witness/cert + explanation

Regulile care “taie” ambiguitatea

  • Vocabular strict: simbol necunoscut = eroare (de ce).
  • DSL @/$: un singur @ pe linie; $ e referință; bare name e vocab (de ce).
  • UNKNOWN este status valid și preferabil unei afirmații neverificabile (de ce).

Ce urmează în capitole

  1. Fundamente booleene și GF(2): de ce XOR/AND (GF(2)).
  2. UBH IR: noduri, hash-consing, semantica (hash-consing).
  3. Solving: SAT + XOR, certificate și explain (SAT, XOR linear).
  4. Limbaje: lexicon/CNL/DSL și typing determinist.
  5. Sesiune și orchestrator: fragments/backends/tactici.
