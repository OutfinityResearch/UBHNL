CNL/DSL Pipeline

Deterministic input (CNL/DSL) → typed IR → fragments/backends → checkable results.

DRAFT: the pipeline is specified as a target. Parsers and compilers are still in progress.
Docs Index Lexicon/CNL/DSL Session API Reasoning Wiki Theory

Stages

  1. Load lexicon + build the symbol table
  2. Tokenize + parse (CNL or DSL)
  3. Typecheck + symbol resolution (strict vocabulary; unknown symbols are errors)
  4. Lower to semantic IR (core logic or a fragment-specific IR)
  5. Fragment classification + backend/tactic planning
  6. Solving + certificate/witness
  7. Check + explain (origins → explanation in CNL/DSL)
