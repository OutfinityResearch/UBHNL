CNL to UBH Pipeline

UBH treats natural language as a controlled, typed input that compiles into Boolean constraints. The pipeline is deterministic and rejects ambiguity.

Pipeline Stages

  1. Lexicon load and symbol table build
  2. Tokenize and parse CNL input
  3. Build a typed AST and validate symbol types
  4. Lower to core logic (forall/exists/and/or/not/implies)
  5. Compile to UBH gates and constraints
