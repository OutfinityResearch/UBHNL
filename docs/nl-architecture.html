CNL to UBH Pipeline

UBHNL treats natural language as a controlled, typed input that compiles deterministically into a typed semantic IR (via DSL), then lowers to the appropriate reasoning fragment (often UBH). The pipeline rejects ambiguity rather than guessing.

Pipeline Stages

  1. Lexicon load and symbol table build
  2. Tokenize and parse CNL input
  3. Build a typed AST and validate symbol types
  4. Lower to core logic (forall/exists/and/or/not/implies)
  5. Lower to fragments/backends (UBH/SAT/SMT/QBF/etc.)
